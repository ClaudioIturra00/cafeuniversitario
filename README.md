# Cafe Universitario

Este proyecto consiste en un sitio web para Café Universitario, un emprendimiento local dedicado a la venta de café, pastelería y productos artesanales.
El sitio permite presentar información de la empresa, mostrar sus productos y registrar solicitudes de información mediante un formulario.

## Secciones del sitio

- Inicio
- Sobre nosotros
- Productos
- Solicitud de información
- Solicitudes registradas
- Footer con información de contacto

## Productos disponibles

- Café Latte
- Muffin de arándanos
- Torta de chocolate
- Berlín relleno

Cada producto incluye una imagen, nombre, descripción y precio.

## Tecnologías utilizadas   

- HTML5
- CSS3
- JavaScript
- GitHub Copilot como herramienta de apoyo
- Geminis (Creacion de imagenes)

## Herramientas de IA como apoyo
Se utilizo IA como apoyo durante el desarrollo del proyecto para generar imagenes, crear textos de contenido y mejorar algunos estilos CSS.
-Prompts utilizados:

- hazme un header, banner de cafe para una cafeteria pagina web, ojala que sean 2 diseños diferentes
- Dame descripciones breves para Café Universitario, Nuestro objetivo y Calidad artesanal.
- Dame descripciones para los productos Café Latte, Muffin de arándanos, Torta de chocolate y Berlín relleno.
- Ayúdame a evitar que el formulario recargue la página al presionar el botón Enviar.
- Crea estilos CSS con colores relacionados con café, crema y chocolate.

## Funcionalidades

- Menú de navegación con enlaces internos.
- Banner principal con enlace hacia los productos.
- Sección informativa sobre Café Universitario.
- Tarjetas para mostrar los productos.
- Formulario de solicitud de información.
- Lista desplegable para seleccionar un producto.
- Visualización inmediata de las solicitudes registradas.
- Diseño responsive para pantallas pequeñas.
- Efectos hover en enlaces, botones y tarjetas.

## Funcionamiento del formulario

- Nombre completo
- Correo electrónico
- Teléfono
- Producto de interés
- Comentario o consulta

Al enviar el formulario, JavaScript agrega automáticamente una nueva fila en la sección “Solicitudes Registradas”, mostrando el nombre, producto seleccionado y correo electrónico.

Los datos se mantienen solamente mientras la página permanece abierta, ya que el proyecto no utiliza una base de datos.
